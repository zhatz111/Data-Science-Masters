Part B

  1. Minimum: This is the smallest value for each feature in the dataset. It gives us a sense of the lower bound of the measurements. For the Iris dataset, the minimum value for each feature can indicate the smallest size of the sepals and petals among all samples.

  2. Maximum: This statistic represents the largest value for each feature. It helps us understand the upper bound of the measurements. In the Iris dataset, the maximum values for sepal and petal length and width can show the largest sizes observed in the sample population.

  3. Mean: The mean is the average value of each feature. It provides a central tendency of the measurements, giving us an idea of the typical size of the sepals and petals in the iris flowers sampled.

  4. Trimmed Mean: The trimmed mean is the mean calculated after "trimming" or removing a certain percentage of the smallest and largest values. It's a measure of central tendency that is less affected by outliers or extreme values. For the Iris dataset, it can give a more robust central value for each feature, especially if there are anomalies or extreme measurements.

  5. Alpha Trimmed Mean: This is a specific type of trimmed mean where the trimming percentage is defined by alpha. For example, a 5% alpha-trimmed mean removes the lowest 5% and the highest 5% of values before calculating the mean. It further refines the idea of a trimmed mean to be more resilient against outliers.

  6. Standard Deviation: This statistic measures the amount of variation or dispersion of the dataset. A low standard deviation indicates that the values tend to be close to the mean, while a high standard deviation indicates that the data is spread out over a wider range of values. In the context of the Iris dataset, it tells us how similar or diverse the sizes of sepals and petals are within each species.

  7. Skewness: Skewness measures the asymmetry of the distribution of values. A skewness value close to 0 indicates a symmetrical distribution. Positive skewness indicates a distribution with a longer tail on the right side, while negative skewness indicates a longer tail on the left. For the Iris dataset, skewness can reveal if the sizes of sepals and petals are evenly distributed or if there are biases towards smaller or larger sizes.

  8. Kurtosis: Kurtosis measures the 'tailedness' of the distribution. A higher kurtosis indicates a distribution with heavier tails and a sharper peak, implying more outliers. A lower kurtosis indicates a flatter distribution. In the Iris dataset, kurtosis can help identify if there are extreme values in the sizes of sepals and petals that deviate significantly from the average.

Based on these statistics, we can draw several conclusions about the Iris dataset:

  • The range (minimum to maximum) gives us an understanding of the overall variability in sepal and petal sizes. Petal_length seems to be the most spread out of all features.
  • The mean and trimmed means provide insight into the central tendency, with trimmed means offering a view less affected by outliers. The mean and trimmed mean differ the most in the two length features meaning some outliers may exist within there data.
  • The standard deviation tells us about the diversity or uniformity in the measurements. The petal_length again shows the most diversity in the data which is aligns with the large range.
  • Skewness and kurtosis offer deeper insights into the shape of the distribution of the data, indicating potential asymmetry and the presence of outliers. Both the petal width and length have similar kurtosis values indicating that they have a flatter distribution with more tailedness.

Class MahalanobisDistance

    Function Initialize(data)
        // Store the data points
        Self.data = data

    Function CalculateCovarianceMatrix()
        // Initialize an empty matrix for covariance, dimensions based on data
        row, col = Dimensions(Self.data)
        covarianceMatrix = ZeroMatrix(col, col)

        // Calculate covariance matrix
        For i from 0 to col-1
            iMean = Mean(Self.data[column=i])
            For j from 0 to col-1
                jMean = Mean(Self.data[column=j])
                // Compute covariance
                covarianceMatrix[i][j] = Sum((Self.data[:,i] - iMean) * (Self.data[:,j] - jMean)) / (row - 1)
        Return covarianceMatrix

    Function CalculateMahalanobisDistance(y)
        // Calculate difference between test point y and mean of data
        diff = y - Mean(Self.data, axis=0)
        covariance = Self.CalculateCovarianceMatrix()
        covarianceInverse = Inverse(covariance)
        // Calculate Mahalanobis distance
        leftMultiplication = Multiply(diff, covarianceInverse)
        mahalanobisDistance = SquareRoot(Multiply(leftMultiplication, Transpose(diff)))
        Return mahalanobisDistance

    Function RemoveOutliers(distances, alpha = 0.05)
        // Determine the threshold for identifying outliers based on the chi-squared distribution
        degreesOfFreedom = NumberOfColumns(Self.data)
        threshold = ChiSquaredCriticalValue(1 - alpha, degreesOfFreedom)
        // Identify outliers
        outliers = distances > SquareRoot(threshold)
        Return outliers

ii. Run Time Analysis

Assumptions:

  1. Data size: Let's assume the input data is an (m \times n) matrix, where (m) is the number of observations (data points) and (n) is the number of features (variables).
  2. Vectorized operations: Operations on numpy arrays (mean, sum, etc.) are generally vectorized and thus can be considered to have linear complexity in relation to the number of elements they process, despite the underlying optimizations that may make them faster in practice.

Complexity of Each Method:

  1. CalculateCovarianceMatrix:

    • Calculating the mean of each column: (O(nm)), as it requires summing over all (m) observations for each of the (n) features.
    • Calculating each element of the covariance matrix involves a sum over (m) elements, repeated for each pair of features, resulting in a complexity of (O(n^2m)).
    • Total: (O(n^2m)), since this term dominates the (O(nm)) term.

  2. CalculateMahalanobisDistance:

    • Calculating the mean and subtracting it from a vector: (O(nm)).
    • Calculating the covariance matrix: (O(n^2m)) as derived above.
    • Inverting the covariance matrix: (O(n^3)), assuming a general matrix inversion is (O(n^3)) where (n) is the dimension of the matrix.
    • Matrix-vector multiplications and the final distance calculation: (O(n^2)).
    • Total: (O(n^2m + n^3)), with matrix inversion being the most computationally expensive step for large (n).

  3. RemoveOutliers:

    • Computing distances for all points involves calculating the Mahalanobis distance for each, leading to (O(m*(n^2m + n^3))).
    • Comparing distances against a threshold is (O(m)).
    • Total: (O(m*(n^2m + n^3))), dominated by the distance calculations for each point.

Total Running Time, (T(n)):

When using the class in a typical manner, where one might calculate the Mahalanobis distance for several points and possibly remove outliers, the total running time depends on how these methods are used together. The most computationally intensive operation is the calculation of the covariance matrix and its inversion, which is part of the Mahalanobis distance calculation. If outliers are removed for (m) points, the running time is:

[T(n) = O(m*(n^2m + n^3))]

from itertools import combinations
import scipy.stats as stats

class mahalanobis_distance:
    
    def __init__(self, data: np.ndarray):
        self.data = data

    # Calculate the covariance matrix using previous assignment code
    def cov_matrix(self):
        row, col = self.data.shape
        cov_mat = np.zeros([col, col])

        for i in range(col):
            i_mean = np.mean(self.data[:,i])
            for j in range(col):
                j_mean = np.mean(self.data[:,j])
                cov_mat[i,j] = np.sum((self.data[:,i] - i_mean) * (self.data[:,j] - j_mean)) / (row - 1)
        return cov_mat

    def mahalanobis(self, y: np.ndarray):
        diff = y - self.data.mean(axis=0)
        covariance = self.cov_matrix()
        cov_inv = np.linalg.inv(covariance)
        left = np.dot(diff, cov_inv)
        mahalanobis_dist = np.sqrt(np.dot(left, diff.T))
        return mahalanobis_dist

    def remove_outliers(self, distances: np.ndarray, alpha: float = 0.05):
        # Degrees of freedom equals the number of features
        degrees_of_freedom = self.data.shape[1]
        # Critical value from chi-squared distribution
        threshold = stats.chi2.ppf((1 - alpha), df=degrees_of_freedom)
        outliers = distances > np.sqrt(threshold)
        return outliers

Function calculate_fisher_score(data):
    Initialize fisher_scores as an empty dictionary

    # Initialize Fisher scores for all columns (features) to zero
    For each column in data.columns:
        fisher_scores[column] = 0

    # Iterate over all unique pairs of columns (features)
    For each pair (i, j) in combinations(data.columns, 2):
        # Calculate mean of each feature in the pair
        mean1 = Calculate mean of data[i]
        mean2 = Calculate mean of data[j]

        # Calculate variance of each feature in the pair
        var1 = Calculate variance of data[i]
        var2 = Calculate variance of data[j]

        # Calculate Fisher score for the pair
        fisher_score = (mean1 - mean2)^2 / (var1 + var2)

        # Add the calculated Fisher score to both features in the pair
        fisher_scores[i] += fisher_score
        fisher_scores[j] += fisher_score

    # Sort the fisher_scores dictionary by values in descending order
    sorted_fisher_scores = Sort fisher_scores by values in reverse (descending) order

    Return sorted_fisher_scores

ii. Run Time Analysis

Assumptions:

  1. Data size: Assume the dataset has (n) samples (rows) and (m) features (columns).
  2. Operation Complexity: Basic numpy operations like mean and variance calculation over an array of size (n) are assumed to have a linear complexity, (O(n)), in terms of the number of elements in the array.
  3. Combination Generation: The number of combinations of (m) features taken 2 at a time is (C(m, 2) = m*(m-1)/2. Generating each combination is considered to have constant time complexity, (O(1)), but iterating over all combinations has (O(m^2)) complexity because the total number of combinations is proportional to (m^2) for large (m).

Complexity Breakdown:

  1. Initializing Fisher Scores: (O(m)) for initializing the Fisher score dictionary for (m) features.

  2. Iterating Over Combinations: (O(m^2)) iterations since we consider each pair of features.

    For each pair of features (each iteration):

    • Mean Calculation: (2*O(n)) since we calculate the mean for each feature separately.
    • Variance Calculation: (2*O(n)) for calculating the variance for each feature.
    • Fisher Score Calculation and Update: (O(1)) for calculating the Fisher score for the pair and updating their scores in the dictionary.

    The operations within each iteration are (4*O(n) + O(1)), simplifying to (O(n)) for operations dominated by (n).

    Thus, the total complexity for this part is (O(m^2*n)).

  3. Sorting Dictionary: Sorting (m) items based on their Fisher score. The complexity of sorting algorithms like Timsort (used in Python) is (O(m log m)) in the worst case.

Total Running Time, (T(n)):

Combining the parts, the total running time of the algorithm is:

[T(n) = O(m) + O(m^2*n) + O(m log m)]

Since (O(m^2*n)) is the dominating term for large datasets with many features, the overall complexity simplifies to:

[T(n) = O(m^2*n)]

from itertools import combinations

def calculate_fisher_score(data):
    fisher_scores = {}
    for i in data.columns:
        fisher_scores[i] = 0
    for i in combinations(data.columns, 2):
        mean1 = np.mean(data[i[0]])
        mean2 = np.mean(data[i[1]])
        var1 = np.var(data[i[0]])
        var2 = np.var(data[i[1]])
        fisher_score = (mean1 - mean2) ** 2 / (var1 + var2)
        fisher_scores[i[0]] += fisher_score
        fisher_scores[i[1]] += fisher_score
    return dict(sorted(fisher_scores.items(), key=lambda item: item[1], reverse=True))

Part C

The complexity analysis is based on the standard 3x3 Tic-Tac-Toe board, meaning there are always 9 spaces and a fixed number of lines (rows, columns, and diagonals) to check.

ConditionalPlayer

  • get_next_move: This method calls get_decisive_move and potentially get_non_decisive_move. The complexity of this method depends on the complexity of the methods it calls.
  • get_decisive_move: Iterates through a fixed number of lines (8 in a standard Tic-Tac-Toe board) and performs constant-time operations within each iteration. Thus, its complexity is (O(1)) because the size of the board (and the number of lines) does not change.
  • get_non_decisive_move: Also performs a fixed number of operations regardless of input size, checking a constant set of corners and then falling back to the first open space. Hence, its complexity is (O(1)).

GoalPlayer

  • Inherits get_next_move and get_decisive_move from ConditionalPlayer, both (O(1)).
  • get_best_move: Similar to get_non_decisive_move in ConditionalPlayer, it iterates through a constant set of spaces and performs a constant-time check for each. Thus, its complexity is (O(1)).

UtilityPlayer

  • Inherits get_next_move and get_decisive_move from ConditionalPlayer, both (O(1)).
  • get_utility_of_lines: Iterates over a fixed set of lines, with constant-time checks and calculations within each iteration. Therefore, its complexity is (O(1)).
  • get_line_utility: Performs constant-time operations on a fixed-size list. Complexity is (O(1)).
  • get_utility_of_spaces: Iterates through a fixed number of spaces (9) and lines (8), performing a constant-time update for each space based on the utility of lines. Although it involves nested iteration, both the number of spaces and the number of lines are constant, leading to (O(1)) complexity.
  • is_line_empty and is_line_full: Both iterate through a fixed-size list (3 elements) and perform constant-time checks, thus each has a complexity of (O(1)).

Summary

All classes perform their operations in constant time, (O(1)), relative to the size of the Tic-Tac-Toe board, because the board size is fixed and does not change. The operations and checks are based on a fixed number of spaces (9) and lines (8), leading to constant time complexities for all methods across the three classes.
